# `/listings` Prompt

```text
Responsive listings browse page for BoilerSub, matching the existing BoilerSub landing page and the full BoilerSub design system.

Goal:
Create the main authenticated marketplace page where verified Purdue students browse available sublease listings. This page should feel premium, editorial, dynamic, and highly scannable, while functioning as the core discovery surface of the product.

Core Function:
Display all active sublease listings in a responsive browse experience with strong hierarchy, listing previews, and clear navigation to view details or create a listing.

Required Elements:
- Persistent authenticated top navigation matching the BoilerSub brand
- BoilerSub wordmark
- Nav links or actions for Browse, List Your Sublease, Profile, and optional guide/help
- A strong editorial page heading like “Available Subleases”
- Supporting copy that reinforces this is a curated Purdue-only marketplace
- Listing grid using reusable listing cards
- Each listing card should include:
  - photo or image placeholder
  - title
  - price
  - lease dates
  - bedrooms / bathrooms
  - address or location snippet
  - optional small status chip like “Available Now”
- Clear CTA near the top: “List Your Sublease”
- Pagination controls at the bottom such as Previous / Next
- Empty state design in case there are no listings
- Optional lightweight filter/search placeholder area that visually reserves space for future advanced features
- Mobile-friendly card layout and touch-first browsing experience

Content Rules:
- The page should feel like a premium student housing marketplace, not a plain CRUD table
- Messaging should emphasize curated, trusted, Purdue-only listings
- Listing cards should be visually rich and easy to scan
- The layout should support future expansion into search, chat, 3D previews, and smarter discovery without needing redesign

Visual Style:
- Must match the BoilerSub landing page design language
- Editorial, premium, kinetic, modern, high-trust
- Use tonal layering instead of hard divider lines
- Use large typography, intentional asymmetry where appropriate, and atmospheric spacing
- Cards should feel elevated, soft, and premium rather than flat commodity listings
- Glassmorphism can be used lightly in top controls or floating actions, but the page should remain product-focused and readable
- Avoid generic dashboard grids or basic marketplace templates

Branding and Colors:
- Off-white / bone background
- Muted Purdue-inspired gold for grounding and secondary emphasis
- Electric blue for primary actions and active states
- Kinetic coral for urgency, featured tags, or energetic accents
- Surface layers should create depth naturally without heavy outlines

Typography:
- Plus Jakarta Sans for page heading, featured prices, and major emphasis
- Manrope for card metadata, labels, supporting text, and controls
- Listing prices and titles should feel bold and visually authoritative

Layout Guidance:
- Desktop: spacious editorial browse layout with strong heading area and a responsive multi-column card grid
- Mobile: stacked cards or compact responsive card grid with excellent readability and tap targets
- The structure should feel premium at both small and large sizes, not just condensed desktop UI

Optional Supporting Detail:
- Include a subtle featured section or highlighted listing treatment at the top
- Reserve visual room for future filters like vibe, budget, or location
- Add a Purdue-context caption or trust statement near the top of the page

Accessibility:
- Strong text contrast
- Clear card hierarchy
- Mobile-friendly touch targets
- Focus states for all interactive elements
- Readable metadata and buttons across breakpoints

Platform:
Responsive web for both desktop and mobile
```
