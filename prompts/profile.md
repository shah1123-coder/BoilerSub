# `/profile` Prompt

```text
Responsive profile page for BoilerSub, matching the full BoilerSub design system and authenticated product experience.

Goal:
Create the current-user profile page where a Purdue student can view and edit their account details. The page should feel trustworthy, polished, and clearly tied to account ownership and marketplace credibility.

Core Function:
Display the current user’s profile information, verification status, and editable fields for personal profile details.

Required Elements:
- Persistent authenticated top navigation
- Strong editorial page heading like “My Profile”
- Profile header area with avatar placeholder, name, and verification status
- Read-only display of email and masked phone number
- Verification badges or status indicators for email / phone / fully verified
- Editable fields for full name and bio
- Primary CTA button: “Save Changes”
- Secondary links or actions for “My Listings” and “Log Out”
- Optional account trust panel reinforcing verified Purdue marketplace participation
- Inline validation, save state, and success/error feedback

Content Rules:
- The page should feel more premium and identity-driven than a generic settings page
- Verification should be clear and confidence-building
- Editing should feel simple and calm
- The design should suggest that profile quality supports marketplace trust

Visual Style:
- Match BoilerSub’s editorial, premium, high-trust visual system
- Use tonal surfaces, atmospheric spacing, and subtle depth
- Avoid generic account settings UIs
- The profile page should feel human, polished, and student-focused

Branding and Colors:
- Off-white / bone background
- Purdue-inspired muted gold for grounding identity cues
- Electric blue for save actions and interactive emphasis
- Kinetic coral only for urgent warnings or important account notices

Typography:
- Plus Jakarta Sans for page heading and key identity emphasis
- Manrope for metadata, labels, helper text, and body content

Layout Guidance:
- Desktop: profile header plus editable details in a premium account layout, potentially with a sidebar action area
- Mobile: stacked identity-first layout with clear edit flow and large controls
- Keep “My Listings” and “Log Out” visible without overwhelming the main page

Optional Supporting Detail:
- Include a short profile trust message such as “Verified BoilerSub members build trust faster”
- Add a compact section previewing marketplace activity if helpful

Accessibility:
- Strong contrast and readable verification states
- Clear labels and save feedback
- Large touch targets for all actions
- Good mobile spacing and form usability

Platform:
Responsive web for both desktop and mobile
```
