# `/listings/[id]` Prompt

```text
Responsive listing detail page for BoilerSub, matching the existing BoilerSub landing page and overall product design system.

Goal:
Create the full detail page for a single sublease listing. This page should feel premium, immersive, and highly trustworthy, giving Purdue students enough detail to evaluate a listing quickly while maintaining the editorial BoilerSub identity.

Core Function:
Display a single listing in full, including pricing, lease dates, location, amenities, description, and owner information, with clear actions for contacting, editing, or deleting depending on user role.

Required Elements:
- Persistent authenticated top navigation consistent with the BoilerSub product
- Large editorial listing header area with title, location, and dominant price presentation
- Photo or visual media area with strong placeholder treatment for future image support
- Lease date range, bedrooms, bathrooms, address, and key details
- Amenities section rendered as branded chips or tags
- Full description block with premium typography and strong readability
- Owner mini-profile card with name, short bio snippet, verification status, and link to public profile
- Primary action area with “Contact Lister” placeholder button
- If current user is owner, include “Edit Listing” and “Delete Listing” actions
- Confirmation-friendly placement for destructive action
- Optional trust strip or microcopy reinforcing verified Purdue-only marketplace

Content Rules:
- The page should feel aspirational and premium, not like a plain property database record
- Price should have strong visual emphasis
- Metadata should be easy to scan
- Owner information should build trust without overwhelming the listing itself
- The layout should leave room for future image galleries, 3D previews, and richer interaction

Visual Style:
- Match the BoilerSub editorial visual system exactly
- Premium, kinetic, modern, layered, high-trust
- Use tonal surfaces, soft depth, and spacing instead of divider lines
- Avoid generic real-estate marketplace page patterns
- Make the listing feel like a curated feature, not just an entry in a catalog

Branding and Colors:
- Off-white / bone base surfaces
- Muted Purdue-inspired gold for grounding accents and subtle metadata emphasis
- Electric blue for key actions and interactive controls
- Kinetic coral for urgency tags or attention moments
- Surface layering should create natural visual hierarchy

Typography:
- Plus Jakarta Sans for title, price, and key callouts
- Manrope for metadata, body copy, labels, and supporting content
- Make the price visually authoritative and memorable

Layout Guidance:
- Desktop: editorial two-column or asymmetrical composition with media and key facts above the fold
- Mobile: stacked immersive layout with strong CTA visibility and readable metadata blocks
- Ensure the page feels premium and intentional across all screen sizes

Optional Supporting Detail:
- Add a small “Available Now” or status chip if appropriate
- Include a subtle sidebar or floating action module on desktop
- Reserve room for future 3D / walkthrough / expanded gallery features

Accessibility:
- Strong contrast and clear hierarchy
- Mobile-friendly CTA sizing
- Clear action separation, especially around delete/edit
- Readable body text and metadata

Platform:
Responsive web for both desktop and mobile
```
